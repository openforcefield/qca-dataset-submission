# OpenFF PEG Ether Fragments TorsionDrives v4.0

### Description

A torsiondrive dataset of small poly(ethylene glycol) (PEG) ether fragments, generated to provide QM reference data for the central `O-CH2-CH2-O` glycol torsion that governs PEG chain conformation. Each molecule is a fragment of a long PEG chain, T-(O-CH2-CH2)n-O-T', sampled across variable length (n = 1-4), end-group registration (a hydroxyl or an sp3 carbon terminus, chosen independently on each end) and an occasional methyl branch. This yields the varied COCCO / COCCOC / CCOCCOC / OCCOCCOC family while always retaining a clean `O-CH2-CH2-O` torsion to drive.

The mapped SMILES and the driven `O-C-C-O` dihedral indices for the molecule set are frozen into the input file `input.json`, which the notebook loads directly; `input.json` is produced by `scripts/generate_fragments.py` (RDKit only). One torsion is driven per molecule: the central glycol `O-CH2-CH2-O` dihedral, preferring a `CX4`-flanked glycol if one exists, otherwise any `O-CH2-CH2-O`. This dataset was generated for a polymer fitting demo.

The intended use of this dataset is to generate QM torsion-energy profiles for the PEG glycol torsion for a polymer fitting demo.

This dataset was generated at the B3LYP-D3BJ/DZVP level of theory using Psi4 with no implicit solvent and default specs. It includes molecules containing the elements C, H, and O, all with a formal charge of 0. The molecular weights range from 62.07 amu to 204.31 amu, with a mean of 146.89 amu.

### QCSubmit generation pipeline

- `generate-dataset.ipynb`: This notebook shows how the dataset was prepared from the input file `input.json`.
- `input.json` is a list of 50 `{"mapped_smiles": ..., "dihedral_indices": [i, j, k, l]}` records, one per molecule. The PEG fragment library is generated by `scripts/generate_fragments.py` (RDKit only): PEG-chain fragments T-(O-CH2-CH2)<sub>n</sub>-O-T' are enumerated across length and end groups, kept only if a glycol `O-CH2-CH2-O` torsion exists, and the first 50 are used. For each molecule, the driven dihedral is the central `O-CH2-CH2-O` glycol torsion (preferring a `CX4`-flanked glycol if one exists, otherwise any `O-CH2-CH2-O`). The mapped SMILES carries atom map numbers = atom index + 1, and the OpenFF `Molecule` is rebuilt with `Molecule.from_mapped_smiles(..., allow_undefined_stereo=True)` so the 0-based `dihedral_indices` line up; the four atoms are verified to be a bonded `O-C-C-O` before the torsion is attached. `input.smi` lists the same 50 molecules as plain (unmapped) canonical SMILES for readability.
- `scripts/generate_fragments.py` is the fragment-generation code (needs only RDKit). Running `python scripts/generate_fragments.py` reproduces both `input.json` and `input.smi` exactly.
- Stereochemistry was not enumerated (`allow_undefined_stereo=True`); tautomers were not enumerated.
- Conformers were generated with `StandardConformerGenerator(max_conformers=5)`.

### Entry / molecule naming

Entry and molecule keys are generated automatically by QCSubmit from the canonical isomeric explicit-hydrogen mapped SMILES; there is no custom naming scheme.

- `dataset.smi`: SMILES for every molecule in the submission.
